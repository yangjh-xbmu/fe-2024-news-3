# 校园跑腿平台 · 产品设计文档

> 日期：2026-05-13 | 状态：待评审

## 一、产品定位与用户角色

**一句话定位**：校园熟人圈的轻量跑腿撮合工具。

**两个角色**：

- **发布者（需求方）**：发布跑腿任务，预付费用到平台。不需要认证即可发单，降低发布门槛。
- **接单者（跑腿方）**：浏览任务、接单、完成、收款。首次接单强制微信手机号授权，此后无需重复。

**角色切换**：同一微信用户既可以是发布者也是接单者。页面底部 Tab 两个入口「发任务」「找任务」，自由切换。账号以微信 openid 唯一标识，角色只是该用户在不同任务中的身份。

**核心数据特征**：
- 每单金额小（5-30 元），决策轻
- 时效性高（买饭 30 分钟内、取快递当天）
- 信任成本低（同校园，物理可追溯）

## 二、核心功能与用户旅程

### 2.1 发任务

1. 选择任务类型：取快递 / 代买饭 / 送文件
2. 填写描述、取送地址（文字）、期望完成时间
3. 设定报酬（建议档位 5/8/10/15 元，可自定义）
4. 预览总费用（报酬 + 平台服务费 5%），确认后微信支付
5. 支付成功 → 任务进入「待接单」状态，费用冻结在平台

### 2.2 接单

1. 进入「找任务」Tab，按类型筛选，按距离/报酬排序
2. 点击「接单」
3. 首次接单弹窗要求授权手机号，之后直接接单
4. 接单成功 → 任务状态变为「进行中」
5. 完成跑腿后点击「完成送达」，上传一张拍照凭证
6. 发布者收到通知，确认后点「确认完成」
7. 费用扣除平台费后自动打入接单者微信零钱

### 2.3 状态流转

```
待接单 → 进行中 → 已送达待确认 → 已完成
  ↓                    ↓
已取消            （24h 无异议自动确认）
```

### 2.4 辅助功能

- **取消**：无人接单时发布者可取消，全额退款。接单后双方均可取消，需填原因
- **评价**：完成后双方互评（👍/👎 + 一句话），展示在个人主页

## 三、数据模型

微信云开发文档数据库。3 个集合：

### users
```js
{
  _openid: "oABC123",           // 微信 openid（内置字段，自动索引）
  phone: "138****1234",         // 微信手机号授权获取
  nickname: "张三",
  avatar: "https://...",
  stats: {                      // 云函数定期计算
    postedCount: 5,
    completedCount: 12,
    cancelledCount: 1
  },
  createdAt: Date,
  updatedAt: Date
}
```

### tasks
```js
{
  _id: "task_xxx",
  publisherId: "oABC123",
  type: "pickup",               // pickup | meal | delivery
  description: "东门中通，取快递，取件码 3-2-5018",
  address: {
    pickup: { name: "东门中通快递点", latitude: null, longitude: null },
    delivery: { name: "北区 12 号楼", latitude: null, longitude: null }
  },
  reward: 800,                  // 报酬，单位：分
  platformFee: 40,              // 平台费，单位：分
  totalFee: 840,                // 总支付金额，单位：分
  expectedBy: Date,             // 期望完成时间
  status: "pending",            // pending | accepted | delivering | delivered | completed | cancelled
  takerId: null,                // 接单者 openid
  paymentRef: {                 // 支付凭证
    prepayId: "wx...",
    transactionId: "420000..."
  },
  timeline: [                   // 关键事件
    { event: "created", at: Date },
    { event: "accepted", at: Date }
  ],
  createdAt: Date
}
```

### reviews
```js
{
  taskId: "task_xxx",
  reviewerId: "oABC123",
  targetId: "oXYZ789",
  rating: 1,                    // 1=👍 0=👎
  comment: "很快，10分钟送到",
  createdAt: Date
}
```

**设计决策**：
- 金额用「分」：避免 JS 浮点精度问题，与微信支付 API 对齐
- task 和 order 合并：一任务只一人接单，不拆分为多个子订单

## 四、技术架构

基于微信云开发（Cloud Base）：

```
┌────────────────────────────────────────────┐
│               微信小程序前端                  │
│   WeUI 组件库 + 原生 WXML/WXSS/JS           │
│   三个 Tab：发任务 / 找任务 / 我的           │
└──────────────┬─────────────────────────────┘
               │ wx.cloud API
┌──────────────┴─────────────────────────────┐
│            微信云开发（Cloud Base）          │
│  云数据库 (JSON 文档)                         │
│  云函数 (Node.js Serverless)                │
│  云存储 (凭证图片)                           │
│  云调用 (微信支付 / 订阅消息 / 手机号)         │
└─────────────────────────────────────────────┘
```

### 云函数拆分

| 云函数 | 职责 | 触发方式 |
|--------|------|----------|
| `payOrder` | 统一下单，返回支付参数 | HTTP 调用 |
| `payCallback` | 支付成功回调，更新任务状态 | 微信支付回调 |
| `handleTask` | 任务增删改查、接单、取消 | HTTP 调用 |
| `handleReview` | 评价创建与查询 | HTTP 调用 |
| `payoutToTaker` | 企业付款到接单者零钱 | 确认完成时调用 |

## 五、支付流

### 5.1 发任务支付

发布者填写任务 → 小程序调 `payOrder` 云函数 → 云函数调 `cloud.cloudPay.unifiedOrder` → 返回 prepay_id → 小程序拉起支付面板 → 支付成功 → 回调创建任务（status: pending）

### 5.2 确认放款

发布者「确认完成」→ `handleTask` 云函数 → 校验状态 → 更新为 completed → 调 `payoutToTaker` → `cloud.cloudPay.unifiedOrder`（企业付款到零钱）→ 95% 进入接单者零钱，5% 留存商户号

### 5.3 异常处理

| 异常 | 处理 |
|------|------|
| 支付成功但回调未到 | 小程序支付成功页直接调 handleTask 建任务（prepay_id 幂等去重） |
| 发布者 24h 不确认 | 定时触发器扫描 delivered 超时任务，自动完成放款 |
| 企业付款失败 | 记录失败，重试 3 次，仍失败通知管理员 |
| 取消退款 | 调微信退款 API 原路退回 |

## 六、信任与安全

- **接单认证**：首次接单强制 `wx.getPhoneNumber` 获取真实手机号
- **手机号脱敏**：138****1234，通过小程序内「联系对方」唤起微信会话
- **任务完成后 24h**：自动切断双方联系方式
- **举报入口**：任务详情页底部，提交后人工审核
- **资金安全**：transactionId 唯一索引防重复支付；每条云函数校验 `wxContext.OPENID` 防越权

## 七、MVP 范围

### MVP 保留
```
发任务（统一下单支付）→ 接单（获手机号）→ 完成送达 →
确认完成（企业付款到零钱）→ 互评
```

### MVP 砍掉
- 信用分系统（靠手机号 + 投诉兜底）
- 订阅消息推送（靠页面内状态刷新）
- 定时自动完成/自动放款（运营初期手工处理）
- 地图选点（文字地址起步）

### 前置准备
- 微信小程序账号（需企业主体或个体工商户，才能开通微信支付）
- 微信云开发开通（按量付费）
- 微信支付商户号（企业主体，T+1 到账，费率 0.6%）
